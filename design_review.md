## `healingeyeclinic` 디자인 검토 및 개발 요구사항

**전반적인 디자인 철학:**
프로젝트는 맞춤형, 고품질, 예술적인 솔루션을 우선시하는 "디지털 장인" 철학을 보여줍니다. 핵심 디자인은 주로 Tailwind CSS v4를 사용하여 구현된 금색 악센트가 있는 "풍부한 시네마틱" 다크 테마입니다. `AgingProcess.tsx`의 정교하게 수작업으로 제작된 SVG 애니메이션은 예술적, 기술적 능력을 보여주는 뛰어난 기능입니다.

---

### 즉시 디자인 수정/수정이 필요한 영역

1.  **접근성: 숨겨진 전역 스크롤바**
    *   **문제점:** 전역 스크롤바가 현재 숨겨져 있어 시각적 스크롤 표시 또는 특정 스크롤 입력 방식에 의존하는 사용자에게 접근성과 유용성을 크게 저해합니다. 이는 기본적인 UX 원칙을 위반합니다.
    *   **권장 사항:**
        *   스크롤바를 숨기기로 한 결정을 재평가하십시오.
        *   만약 커스텀 스타일링을 원한다면, 시각적으로 명확하고 사용자 접근성 설정을 존중하며 스크롤 가능한 콘텐츠를 명확하게 나타내는 커스텀 스크롤바 스타일을 구현하십시오. 이러한 커스텀 스타일이 크로스 브라우저 호환성을 갖도록 하십시오.

---

### 추가 디자인 개발이 필요한 영역

1.  **포괄적인 디자인 시스템 / UI 키트 개발**
    *   **현재 상태:** `src/components/ui` 디렉토리가 부족하며, 이는 중앙 집중식의 포괄적인 UI 키트가 부족함을 나타냅니다. `button.tsx`와 같은 개별 구성 요소는 고품질이지만, 이러한 맞춤형 접근 방식은 프로젝트가 확장됨에 따라 디자인 불일치 및 유지 관리 문제를 초래할 수 있습니다.
    *   **권장 사항:** `src/components/ui` 내에서 UI 키트를 확장하고 공식화하십시오.
        *   **구성 요소 표준화:** 다음을 포함하되 이에 국한되지 않는 더 광범위한 재사용 가능하고 일관된 UI 구성 요소를 개발하십시오: 입력 필드, 텍스트 영역, 확인란, 라디오 버튼, 드롭다운(선택), 카드 구성 요소, 모달, 경고/토스트, 페이지 매김 컨트롤 및 탭.
        *   **디자인 토큰:** Tailwind CSS v4의 기능 또는 CSS 변수를 사용하여 완전한 디자인 토큰 세트(색상 팔레트, 타이포그래피 스케일, 간격 단위, 테두리 반경, 그림자, 중단점)를 명확하게 정의하고 문서화하며, 필요한 경우 `globals.css`를 넘어 확장하십시오.
        *   **변형 일관성:** 모든 UI 구성 요소에 걸쳐 구성 요소 변형(예: 기본, 보조, 고스트 버튼; 다양한 카드 스타일)을 정의하고 적용하기 위한 명확한 패턴 및 지침을 설정하여 균일성을 보장하십시오.

2.  **향상된 시각적 일관성 ("와우" 순간을 넘어)**
    *   **현재 상태:** 이 프로젝트는 영향력 있고 내러티브 중심의 시각적 경험(예: `AgingProcess.tsx`)을 만드는 데 탁월합니다. 그러나 일관된 사용자 경험을 보장하기 위해 모든 표준 UI 요소 및 콘텐츠 레이아웃에 대한 디자인 원칙의 일관된 적용에 더 많은 주의를 기울여야 합니다.
    *   **권장 사항:**
        *   **디자인 감사:** 모든 기존 페이지 및 섹션에 대한 철저한 디자인 감사를 수행하여 간격, 타이포그래피 사용, 시각적 계층, 정렬 및 구성 요소 적용의 불일치를 식별하고 수정하십시오.
        *   **콘텐츠 표현 지침:** 다양한 콘텐츠 유형(기사, 블로그 게시물, 서비스 설명, 팀 프로필)을 표시하기 위한 특정 디자인 지침을 개발하여 최적의 가독성, 시각적 매력 및 "시네마틱" 테마와의 일치를 보장하십시오. 여기에는 이미지 및 비디오 통합 규칙(종횡비, 크기 조정, 배치)이 포함됩니다.

3.  **정교한 타이포그래피 시스템**
    *   **현재 상태:** 사용자 지정 글꼴이 사용되지만, 더 공식적이고 계층적인 타이포그래피 시스템은 가독성과 시각적 구조를 향상시킬 수 있습니다.
    *   **권장 사항:**
        *   제목(H1-H6), 본문 텍스트(단락, 목록), 캡션 및 유틸리티 텍스트에 대한 명확한 타이포그래피 스케일을 정의하십시오.
        *   다양한 화면 크기 및 콘텐츠 블록에서 가독성을 최적화하기 위해 줄 높이, 글자 간격 및 단락 간격에 대한 규칙을 설정하십시오.

4.  **마이크로 인터랙션 및 페이지 전환**
    *   **현재 상태:** `framer-motion`은 이미 특정 요소(예: 버튼, `AgingProcess.tsx`)에 대한 정교한 애니메이션에 활용됩니다.
    *   **권장 사항:**
        *   **일관된 마이크로 인터랙션:** 일반적인 사용자 작업(예: 양식 제출, 탐색 클릭, 상태 변경)에 대한 일관된 마이크로 인터랙션 세트를 정의하여 미묘한 피드백을 제공하고 인식된 반응성을 향상시키십시오.
        *   **글로벌 페이지 전환:** `framer-motion` 또는 유사한 라이브러리를 활용하여 애플리케이션의 다른 경로 간에 더 부드럽고 시네마틱한 흐름을 만들기 위해 미묘하고 테마에 적합한 페이지 전환을 구현하십시오.

5.  **시각적 요소에 대한 성능 최적화**
    *   **현재 상태:** 고도로 상세한 SVG 애니메이션 및 `framer-motion`을 복잡한 효과에 사용하면 세심하게 최적화되지 않으면 성능에 영향을 미칠 수 있습니다.
    *   **권장 사항:**
        *   디자인 및 개발 워크플로에 성능 고려 사항을 직접 통합하십시오.
        *   특히 저사양 장치에서 모든 애니메이션이 부드러운 60fps 성능을 위해 최적화되었는지 확인하십시오.
        *   초기 페이지 로드 시간을 개선하기 위해 복잡한 시각적 자산 및 애니메이션에 대해 적절한 경우 지연 로딩을 구현하십시오.
        *   정기적으로 성능을 프로파일링하여(예: 브라우저 개발 도구 사용) 시각적 요소로 인해 발생하는 병목 현상을 식별하고 해결하십시오.
